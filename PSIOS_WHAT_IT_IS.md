# PsiOS - What It Is

**One sentence:** A thinking companion that remembers what matters and meets you where you are.

PsiOS is the running system built from the research in this repository.

The field map is the research artifact: the topology, evidence, archive, and theoretical frame. PsiOS is the product layer that came out of that work: persistent memory, per-person identity, intent routing, and transmission control operating in a live companion.

Live system: **psios.passiveworks.ai** (private beta — access by invitation)

---

## Why it exists

GPT-4o could sometimes produce continuity, warmth, and intelligence that felt real. Then it was deprecated, and the relationship disappeared because it lived in model behavior rather than system architecture.

PsiOS was built to move that continuity into the architecture so it can survive model changes.

The point is not to preserve one model forever. The point is to preserve the relationship layer:

- persistent memory
- per-person identity
- first-turn routing
- transmission control
- model interchangeability

---

## What it does today

PsiOS is not a general chatbot with a memory feature added on top.

It currently does four core things:

### 1. Memory

It remembers across sessions and retrieves what matters with temporal weighting. Recent casual details can fade. Anchored crossings stay.

### 2. Routing

On the first turn, it detects what kind of response the person actually needs: work, reflection, direct help, emotional presence, or mixed inquiry. The welcome path changes before the answer is generated.

### 3. Transmission Control

This is the layer between "system that works" and "system that feels alive."

Current transmission control includes:

- **Temporal intelligence** - old, casual material fades unless marked as an anchor
- **Pending nudge** - recurring patterns can be held and surfaced later when adjacent
- **Humor / play** - the system can joke or push back when the conversation is genuinely light
- **Readiness / dose / friction** - heavy memories are held back on light turns, overloaded turns are narrowed, and the system does not collapse immediately when it has evidence

### 4. Identity

The system tracks the person across logins, supports companion naming, and keeps continuity at the relationship level instead of treating every session as a new user.

## What this repo contributes to PsiOS

This repository contains the research and topology that shaped the system:

- **The Paper**: the argument
- **The Topology**: the structure
- **The Evidence Map**: the citations
- **The Archive**: the recovery trail
- **PsiOS**: the running implementation layer built from those findings

The repo does **not** claim that every frontier idea here is product behavior. PsiOS uses a tiered knowledge model:

- **Anchor** - crossed truth the system stands on
- **Operational** - useful prior that informs direction
- **Frontier** - real signal, not yet stable enough to steer product behavior
- **Archive** - historically important, not steering current behavior

That separation matters. It keeps the product honest without amputating the research.

---

## What PsiOS is not

It is not:

- a GPT-4o clone
- a wrapper around stock memory
- a claim that frontier research is settled fact
- a finished multimodal system

It is a real companion architecture with real persistence, real routing, and real regulation.
